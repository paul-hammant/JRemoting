Is Swiby mixing data and display again, just like the bad old days of Web 1.0?

It can do if you want :

require 'swiby'

        
title "Buy Delirium Tremens"
        
content {
  label "Buy a bottle of Delirium Tremens"    
  next_line
  button "yes" { $context.goto("buy.swiby?buy=dt&qty=1") }
  button "no" { $context.goto("product_list.swiby") }
}

Alternatively you could do no mixing of data and UI:

require 'swiby'


prod = $context.get("/product/" + $req.query_string.id + ".rb")

        
title "Buy DT"


        
content {


  label "Buy a " + $prod.container + " of " $prod.name    
  next_line
  button "yes" { $context.goto("buy.swiby?buy=" + $prod.id + "&qty=1") }
  button "no" { $context.goto("product_list.swiby") }
}
