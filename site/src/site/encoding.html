Overview

JRemoting has three stream encoding implementations - ByteStream, ObjectStream and XmlStream.  ByteStream is the default and slightly faster on the wire than ObjectStream.  Both are binary encodings.  XmlStream leverages XStream for serializing  text based.

Server side usage

server = new SocketServer(new ConsoleServerMonitor(), new ByteStream(), new InetSocketAddress(10333));

Client side usage

jremotingClient = new JRemotingClient(new SocketTransport(new ConsoleClientMonitor(), new ByteStream(), new InetSocketAddress("localhost", 10333)));
To use either of XmlStream or ObjectStream, use them in place of ByteStream when setting up the client and server sides.  

Encodings mush be matched, of course, on both sides to work at all.  JRemoting will fail, not necessarily quickly or clearly when you've mismatched encodings.
